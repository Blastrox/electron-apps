Architecture

Deux processus,
un seul exécutable

Electron embarque Chromium pour l'interface et Node.js pour le système dans un seul binaire. Ils ne se parlent jamais directement.

⚙️

Main Process

main.js

  • Démarre l'application
  • Crée les fenêtres
  • Accède aux fichiers & DB
  • Un seul dans toute l'app
  • Analogie : serveur Express
🔒

Preload

preload.js

  • S'exécute avant le renderer
  • Accès à ipcRenderer
  • Expose une API contrôlée
  • Le renderer ne voit jamais ipcRenderer
  • Analogie : sas de sécurité
🖥️

Renderer Process

renderer/index.html + app.js

  • Affiche l'interface (Chromium)
  • Gère les clics et le DOM
  • Un par fenêtre
  • Jamais accès à Node.js
  • Analogie : frontend Angular

Flux de données

Comment ils se parlent

IPC (Inter-Process Communication) — comme un appel HTTP, mais en local.

renderer/app.js └─ window.counterService.increment() │ (exposé par contextBridge) ▼ preload.js └─ ipcRenderer.invoke('counter:increment') │ (traverse le pont sécurisé) ▼ main.js └─ ipcMain.handle('counter:increment', () => ++count) │ (retourne la valeur) ▲ preload.js └─ Promise résolue ▲ renderer/app.js └─ const count = await window.counterService.increment()
